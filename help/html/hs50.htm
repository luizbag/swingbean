Tipos de Componente

Seguem abaixo os tipos de componentes existentes no framework:
  • TEXT - uma caixa de texto (default para campos do tipo String)
  • COMBO - um combo box  (default se possuir um dos atributos comboModelClass ou comboList)
  • DATE - um componente de calendário (default para campos do tipo Date)
  • INTEGER - uma caixa de texto que aceita apenas inteiros (default para campos do tipo int)
  • BOOLEAN - um checkbox (default par ao tipo boolean)
  • DOUBLE - uma caixa de texto que aceita apenas decimais (default para campos do tipo double)
  • FLOAT - uma caixa de texto que aceita apenas decimais (default para campos do tipo float)
  • LONG - uma caixa de texto que aceita apenas inteiros (default para campos do tipo long)
  • LARGE_TEXT* - uma caixa grande de texto para entrada de String. Deve ser configurada explicitamente.
  • PASSWORD* - uma caixa de texto com máscara. Deve ser configurada explicitamente.
  • DEPENDENT_COMBO - um combo box que depende de outro (default se possuir o atributo dependentProperty)
  • MULTIPLE_LIST* - duas listas com "drag and drop" para arrays de objetos (default se possuir um dos atributos listModelClass ou list)
  • CHECKBOX_LIST* - uma lista de checkboxes para arrays de objetos. Deve ser configurada explicitamente.
  • IMAGE** - uma lista para entrada de imagens em array de bytes (default se for array de bytes)
  • COLOR - um componente de escolha de cores (default para campos do tipo Color)
    ·
    * Estes campos não são adequados para tabelas
    ** Este campo na tabela deve ser configrado como readOnly e não pode ser editável.
